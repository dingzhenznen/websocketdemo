  TCP/IP 协议栈分层

  ┌─────────────────────────────────────┐
  │         应用层 (HTTP/FTP/DNS)        │  ← 你的代码操作这一层
  │         net.createServer()          │
  ├─────────────────────────────────────┤
  │         传输层 (TCP/UDP)             │  ← 负责：端到端可靠传输
  │    TCP: 连接管理、流量控制、重传机制   │
  ├─────────────────────────────────────┤
  │         网络层 (IP)                  │  ← 负责：寻址和路由
  │    IP: 逻辑地址、路由选择、分组转发     │
  ├─────────────────────────────────────┤
  │         链路层 (以太网/WiFi)          │  ← 负责：物理寻址、帧传输
  │    MAC地址、ARP、帧封装               │
  ├─────────────────────────────────────┤
  │         物理层 (网线/无线信号)         │  ← 负责：比特流传输
  └─────────────────────────────────────┘

  各层职责详解

  各层职责详解

  1. 传输层 (TCP) - "负责传输"

  ┌──────────┬────────────────────────────────────┐
  │   功能   │                说明                │
  ├──────────┼────────────────────────────────────┤

  ├──────────┼────────────────────────────────────┤
  │ 可靠传输 │ 确认机制(ACK)、超时重传、去重      │
  ├──────────┼────────────────────────────────────┤
  │ 流量控制 │ 滑动窗口，防止发送过快压垮接收方   │
  ├──────────┼────────────────────────────────────┤
  │ 拥塞控制 │ 慢启动、拥塞避免，防止网络拥塞     │
  ├──────────┼────────────────────────────────────┤
  │ 端口寻址 │ 通过端口号区分不同应用进程         │

  │ 分片重组 │ 大数据包分片传输，目的地重组       │
  └──────────┴────────────────────────────────────┘

  IP 头部: [源IP地址] [目的IP地址] [TTL生存时间] [协议类型] ...

  数据封装过程（发送时）

  应用数据: "hello\r\n"
      ↓
