 1. 浏览器原生支持

  ┌────────────┬─────────────────┬───────────────────────┐
  │            │    WebSocket    │ 原生 Socket (TCP/UDP) │
  ├────────────┼─────────────────┼───────────────────────┤
  │ 浏览器     │ ✅ 原生支持     │ ❌ 不支持             │
  ├────────────┼─────────────────┼───────────────────────┤
  │ JavaScript │ new WebSocket() │ 需要 WebRTC 或插件    │
  ├────────────┼─────────────────┼───────────────────────┤
  │ 跨平台     │ 所有现代浏览器  │ 需要额外封装          │
  └────────────┴─────────────────┴───────────────────────┘

  // WebSocket - 浏览器直接可用
  const ws = new WebSocket('ws://localhost:8080');

  // 原生 Socket - 浏览器里无法直接使用
  // 只能在 Node.js/Deno/原生应用中使用
  const net = require('net'); // Node.js only

  3. 自动处理粘包/拆包

  ┌──────────┬─────────────────────┬────────────────────┐
  │          │      WebSocket      │ 原生 Socket (TCP)  │
  ├──────────┼─────────────────────┼────────────────────┤
  │ 数据边界 │ 帧(Frame)有明确边界 │ 字节流，无边界     │
  ├──────────┼─────────────────────┼────────────────────┤
  │ 粘包处理 │ 自动处理            │ 需要手动实现       │
  ├──────────┼─────────────────────┼────────────────────┤
  │ 数据类型 │ 文本/二进制自动区分 │ 全是字节，自己解析 │
  └──────────┴─────────────────────┴────────────────────┘

  // WebSocket - 按帧接收，自动处理边界
  ws.onmessage = (event) => {
    console.log(event.data); // 完整的一条消息
  };

  // 原生 Socket - 可能收到粘包数据
  socket.on('data', (data) => {
    // 可能收到: "消息1的一半消息2的一半"
    // 需要自己实现分包逻辑
  });

  4. 内置心跳机制

  // WebSocket 内置 Ping/Pong 帧（协议层）
  // 浏览器/服务器自动处理心跳

  // 原生 Socket - 必须自己实现
  function heartbeat() {
    socket.write('ping'); // 自定义心跳包
    setTimeout(heartbeat, 30000);
  }

  8. 适用场景对比

  选择 WebSocket 当：

  - 开发 Web 应用（聊天、通知、实时数据）
  - 需要 快速开发，不想处理底层细节
  - 要穿透 企业防火墙
  - 需要 浏览器客户端

  实时聊天    ✅ WebSocket
  股票行情    ✅ WebSocket
  在线协作    ✅ WebSocket
  直播弹幕    ✅ WebSocket

  选择原生 Socket 当：

  - 开发 游戏服务器（追求极致性能）
  - IoT 设备通信（资源受限）
  - 微服务内部通信
  - 需要 自定义协议

  MMORPG游戏   ✅ 原生 Socket (或 UDP)
  视频流传输   ✅ 原生 Socket (或 WebRTC)
  物联网设备   ✅ 原生 Socket/MQTT
  高频交易     ✅ 原生 Socket

  9. 代码对比

  WebSocket（简洁）

  // 客户端（浏览器）
  const ws = new WebSocket('ws://localhost:8080');
  ws.onopen = () => ws.send('Hello');
  ws.onmessage = (e) => console.log(e.data);

  // 服务器（Node.js + ws库）
  const WebSocket = require('ws');
  const wss = new WebSocket.Server({ port: 8080 });
  wss.on('connection', (ws) => {
    ws.on('message', (msg) => ws.send(msg));
  });

  原生 Socket（繁琐）

  // 服务器
  const net = require('net');
  const server = net.createServer((socket) => {
    let buffer = ''; // 需要手动处理粘包

    socket.on('data', (data) => {
      buffer += data.toString();
      // 自己找消息边界
      let lines = buffer.split('\n');
      buffer = lines.pop();
      lines.forEach(line => handleMessage(line));
    });
  });
  server.listen(8080);

  一句话总结：
  
  ▎ WebSocket = 方便 + 浏览器支持 + 自动处理很多问题
  ▎
  ▎ 原生 Socket = 灵活 + 高性能 + 需要处理所有细节

  Web 开发首选 WebSocket，性能敏感或特殊场景用原生 Socket
